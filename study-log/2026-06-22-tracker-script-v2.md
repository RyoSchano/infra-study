## 設計
```
変数
├ LOG_DIR       → ログディレクトリ
├ TODAY         → 今日の日付
├ LOG_FILE      → 今日のログファイル
└ LAST_STATUS   → 最終状態（IN/OUT）

引数 in|out|show
├ in    →   作業開始入力
│   ├   ファイルがない  → 記録
│   ├   最終状態OUT    → 記録
│   └   最終状態IN     → エラー
│
├ out   →   作業終了入力
│   ├   ファイルがない  → エラー
│   ├   最終状態IN     → 記録
│   └   最終状態OUT    → エラー
│
├ show  →   ログ表示
│   └   ファイルがない  → エラー
│
└ *     →   使い方表示

```
## 学んだこと
- $NF Number of Fields (列数)
- 変数には "" を付けるようにしよう  
    変数が空文字になる場合やスペースを含む場合に意図しない動作を防ぐため 
- LAST_STATUS=$(awk ~~~) はファイル存在チェックより前に実行するダメ  
    awkが動いてしまうから初回ファイルない時にエラーになる
- アルゴリズム作成は設計図から作り始めること

#### awk
- BEGIN  
    ファイルを読む前に一回だけ実行
- END  
    ファイルを全部読み終わった後に最後に読んだ行を使う

## つまずいたこと
- LAST_STATUS=$(awk 'END {print $NF}' "$LOG_FILE") を入れるところ
    - 原因  
    caseの外に入れると awk が働いてしまうのでファイルのない初回 in はエラーになる
    - 解決方法  
    設計図を確認しながら各 case 分岐先に if 文を挿入
- show の cat はファイルがないとエラーになるのでファイル存在確認する
- 引数を IN で受け取った場合のみに対して in で入力していたため重複可能になっていた
