# tracker.sh

## 1. 目的・背景
- 作業開始・終了時刻をログに記録し、1日の作業時間を集計する

## 2. 使い方・仕様（インタフェース）
```
実行例: tracker.sh < in|out|show|summary >
*引数・入力
├ in        → 開始時刻
├ out       → 終了時刻
├ show      → 今日のログを表示
└ summary   → 今日の作業時間を集計

*保存・出力
├ ログ保存場所: $HOME/tracker-logs/YYYY-MM-DD-tracker.log
└ ログ形式: YYYY-MM-DD HH:MM:SS <in|out>
```
## 3. 処理フロー（メインロジック）
【大枠の処理の流れ】
↓ ログ保存先ディレクトリなければ作成
↓ 引数を受け取り分岐処理
### in
1. 今日のログファイル存在確認
2. ファイルがあれば最終ステータスを調べる
3. ステータスが IN ならエラー
4. OUT なら現在時刻と IN を追記

### out
1. 今日のログファイル存在確認
2. なければエラー
3. あれば最終ステータスを調べる
4. OUT ならエラー
5. IN なら現在時刻と OUT を追記

### show
1. 今日のログファイル存在確認
2. ないならエラー
3. 今日のログファイルを表示

### summary
1. 今日のログファイル存在確認
2. ないならエラー
3. あれば最終ステータスを確認
4. IN ならエラー
5. OUT なら集計
6. 今日の活動時間の集計を表示

## 4. 内部設計（使用する値）
```
*主要変数・定数
├ LOG_DIR   → ログファイル保存先ディレクトリ
├ TODAY     → 今日の日付
└ LOG_FILE  → 日付付きファイル名

```
## 5. エラー想定・例外処理
【発生しそうなエラーとその時の挙動】
- 引数の大文字小文字判定    → tr で解決
